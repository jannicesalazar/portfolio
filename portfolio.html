Touch-Based Coding App

Project Type: Research study + mobile app prototype
My Role: UX/UI Designer (front-end design and prototyping)
Team: Collaborated with a classmate responsible for backend development

This project explored the design of a mobile app that allows users to code without a computer, using different interaction methods to understand which is the most efficient and intuitive for beginners.

We specifically designed and tested three coding interaction models:

Python (typing method): traditional text-based coding using a keyboard
Scratch (dragging method): block-based coding using drag-and-drop interactions
Touch method (our design): a coding system designed for mobile devices

The main focus of the study was to compare these three approaches in terms of task completion speed and usability, especially in a mobile-first environment where traditional coding methods are less accessible.

Goal:
To determine whether a touch-based coding interface could allow users to code faster and more intuitively than typing (Python) and dragging (Scratch), particularly in a mobile app environment.

Target Users:
Beginner or aspiring programmers who want to learn or write code but may not have access to a computer or find traditional coding environments difficult to use.

Tools Used:
Figma (UI/UX design and interactive prototyping)
Google Forms (user testing and feedback collection)
Google Sheets (organizing and analyzing test results)

Problem

Coding is traditionally designed around structured input methods such as typing or drag-and-drop interactions. While these approaches are effective in desktop environments, they can still present challenges for beginners in terms of speed, usability, and ease of interaction—especially when transitioning to mobile or more flexible learning environments.

Python (text-based coding) requires users to type syntax accurately, which can slow down beginners and increase cognitive load. Scratch (drag-and-drop coding) reduces syntax barriers by allowing users to visually construct logic, but it can still feel slow and less fluid for rapid task completion due to repeated dragging and arranging of blocks.

This creates an opportunity to explore alternative interaction models that may better support speed and intuitiveness in coding tasks.

This project investigates whether a touch-based coding interface within a mobile app environment can improve task completion speed and usability compared to traditional typing (Python) and drag-and-drop (Scratch) methods.

Research and Methodology

To evaluate speed and usability across different coding interaction methods, I designed and tested three interactive prototypes:

Block-based prototype (Scratch-style): drag-and-drop system
Text-based prototype (Python-style): traditional coding interface
Touch-based prototype (final design): mobile-first touch interaction model

All prototypes were designed to test the same core programming tasks while varying only the interaction method.

The touch-based coding prototype was informed by Python’s logical structure, but redesigned for mobile interaction.

Ethical Approval:
An IRB proposal was submitted and approved prior to testing.

Testing Approach:
Participants completed identical coding tasks using each prototype.

Data Collection included task completion time, usability feedback, and observational notes.

Reflection

This project helped me understand the importance of iteration, feedback, and designing for accessibility and ease of use.
